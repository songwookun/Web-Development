# Web-Development

**안보라 국어 학원** 웹사이트 프로젝트입니다.
HTML, CSS, JavaScript만으로 구축한 SPA(Single Page Application)로, 별도 서버 없이 정적 호스팅만으로 운영할 수 있습니다.

> **Live Demo:** https://songwookun.github.io/Web-Development/

---

## 프로젝트 구조

```
Web/
├── index.html        # 메인 HTML (SPA 진입점)
├── css/
│   └── style.css     # 전체 스타일시트 (반응형 포함)
├── js/
│   ├── data.js       # 기본 데이터 및 localStorage CRUD 함수
│   ├── admin.js      # 관리자 인증 및 편집 기능
│   └── app.js        # SPA 라우터 및 페이지 렌더링
└── img/
    └── logo.png      # 학원 로고 이미지
```

## 주요 기능

| 기능 | 설명 |
|------|------|
| **SPA 라우팅** | Hash 기반(`#home`, `#instructors` 등) 라우팅으로 페이지 새로고침 없이 화면 전환 |
| **배너 슬라이더** | 복수 배너 추가 가능, 2개 이상이면 5초 간격 자동 슬라이드 |
| **강사진 소개** | 강사 프로필 카드 (이름, 직책, 소개, 사진, 외부 소개 링크) |
| **교육과정** | 과정별 상세 정보 (대상, 수업 시간, 설명) |
| **입학 테스트** | 테스트 절차 안내 및 상세 정보 |
| **공지사항** | 목록/상세 보기, 검색 기능 지원 |
| **관리자 모드** | 비밀번호 인증 후 모든 콘텐츠를 인라인으로 추가/수정/삭제 가능 |
| **반응형 디자인** | 데스크탑/태블릿/모바일 모두 지원 (768px 이하 햄버거 메뉴) |
| **플로팅 연락처 버튼** | 전화번호, 카카오톡, 네이버 카페 등 다양한 연락 수단 지원 |
| **약도/약관/개인정보 편집** | 캠퍼스 정보, 이용약관, 개인정보처리방침 모두 편집 모드에서 수정 가능 |

## 실행 방법

### 방법 1: 파일 직접 열기
별도의 설치나 빌드 과정이 필요 없습니다.

```bash
# 레포지토리 클론
git clone https://github.com/songwookun/Web-Development.git
cd Web-Development

# index.html을 브라우저에서 열기
open index.html        # macOS
start index.html       # Windows
xdg-open index.html    # Linux
```

### 방법 2: 로컬 서버로 실행
```bash
# Python이 설치되어 있다면
python3 -m http.server 8080

# Node.js가 설치되어 있다면
npx serve .
```
이후 브라우저에서 `http://localhost:8080` 으로 접속합니다.

## 관리자 모드 사용법

1. 페이지 하단 푸터의 **저작권 문구**(© 2026 안보라...)를 **3회 빠르게 클릭**
2. 비밀번호 입력 창이 표시되면 관리자 비밀번호 입력
3. 상단에 초록색 **편집 모드** 바가 나타나면 활성화 완료
4. 각 콘텐츠 카드에 표시되는 **수정/삭제** 버튼으로 편집
5. **편집 종료** 또는 **로그아웃** 버튼으로 편집 모드 해제

> 관리자 비밀번호는 SHA-256 해시로 저장되어 있습니다.
> 변경하려면 `js/admin.js`의 `ADMIN_PW_HASH` 값을 교체하세요.
> 해시 생성 방법은 해당 파일 상단 주석을 참고하세요.

## 편집 가능한 항목

편집 모드에서 수정할 수 있는 모든 항목입니다.

### 배너 (메인 슬라이더)
- **추가/수정/삭제** 가능
- 각 배너: 강조 문구, 이어지는 문구, 설명, 배경 이미지 URL, 버튼 링크 URL, 버튼 표시 이름
- 버튼 링크 URL과 표시 이름을 모두 입력해야 버튼이 표시됨 (비우면 버튼 숨김)
- 배너가 2개 이상이면 5초 간격으로 자동 슬라이드, 하단 인디케이터 점으로 수동 이동 가능

### 강사진 소개
- **추가/수정/삭제** 가능
- 각 강사: 이름, 직책, 소개, 이미지 URL, 소개 링크 URL, 링크 표시 이름
- 소개 링크를 입력하면 카드에 링크 버튼이 표시됨 (표시 이름 미입력 시 "더 보기"로 표시)
- 이미지 URL을 비우면 플레이스홀더 이미지가 자동 표시됨

### 교육과정
- **추가/수정/삭제** 가능
- 각 과정: 과정명, 구분(태그), 설명, 대상, 수업 시간

### 입학 테스트
- **테스트 절차 단계**: 추가/수정/삭제 가능
- **테스트 안내 정보**: 일정, 소요시간, 영역, 준비물, 비용, 문의 전화, 구글폼 링크

### 공지사항
- **추가/수정/삭제** 가능
- 각 공지: 제목, 날짜, 내용, 중요 공지 여부
- 목록 페이지에서 검색 기능 지원

### 연락처 (플로팅 버튼)
- **추가/수정/삭제** 가능
- 각 연락처: 이름, 전화번호(선택), 링크 URL(선택), 링크 표시 이름(선택)
- 전화번호가 있으면 클릭 시 전화 연결
- 전화번호가 없고 링크만 있으면 클릭 시 해당 링크로 이동
- 기본: 본관(전화), 카카오톡 상담(링크), 네이버 카페(링크)

### 약도 (오시는 길)
- **캠퍼스 정보**: 추가/수정/삭제 가능 (이름, 라벨, 주소, 참고사항, 지도 검색어)
- **대중교통/운영시간**: 수정 버튼으로 교통 안내 텍스트, 운영 시간 편집 가능
- 카카오맵 연동: 각 캠퍼스의 지도 검색어를 기반으로 카카오맵 링크 자동 생성

### 사이트 이용약관
- 각 조항을 **항목별로 추가/수정/삭제** 가능
- 각 항목: 제목, 내용
- 내용에서 `- `(하이픈+공백)으로 시작하는 줄은 목록으로 자동 렌더링

### 개인정보처리방침
- 이용약관과 동일한 방식으로 **항목별 추가/수정/삭제** 가능

## 이미지 등록 방법 (Google Drive 활용)

강사 사진, 배너 배경 이미지 등에 외부 이미지 URL을 사용할 수 있습니다.
Google Drive를 이미지 호스팅으로 활용하는 방법은 다음과 같습니다.

### 1단계: Google Drive에 이미지 업로드
- Google Drive에 접속하여 이미지 파일을 업로드합니다.

### 2단계: 공유 설정 변경
1. 업로드한 이미지 파일을 **우클릭** → **공유** 클릭
2. 공유 창 하단의 **"제한됨"** 부분을 클릭
3. **"링크가 있는 모든 사용자"** 로 변경 (뷰어 권한)
4. **"링크 복사"** 클릭

### 3단계: 링크 변환
복사된 링크는 아래와 같은 형태입니다:
```
https://drive.google.com/file/d/FILE_ID/view?usp=sharing
```

`/file/d/` 와 `/view` 사이에 있는 부분이 **파일 ID**입니다.

이 ID를 아래 형태의 URL에 넣어 변환합니다:
```
https://lh3.googleusercontent.com/d/FILE_ID
```

### 변환 예시

| 구분 | 값 |
|------|-----|
| **복사한 링크** | `https://drive.google.com/file/d/10BUeRk98TdGRKN8mYiAfZ0li8gKfrQR-/view?usp=sharing` |
| **파일 ID** | `10BUeRk98TdGRKN8mYiAfZ0li8gKfrQR-` |
| **사용할 URL** | `https://lh3.googleusercontent.com/d/10BUeRk98TdGRKN8mYiAfZ0li8gKfrQR-` |

> **주의:** 파일 ID에 하이픈(`-`)이나 언더스코어(`_`) 같은 특수문자가 포함될 수 있습니다. ID를 빠뜨리지 않고 정확히 복사해야 합니다.

### 4단계: URL 입력
변환된 URL을 편집 모드에서 이미지 URL 필드에 붙여넣습니다.
- 강사 사진: 강사 수정 → "이미지 URL" 필드
- 배너 배경: 배너 수정 → "배경 이미지 URL" 필드

### 다른 이미지 호스팅 서비스
Google Drive 외에도 아래 서비스를 사용할 수 있습니다:
- **[imgbb.com](https://imgbb.com)** - 무료 이미지 업로드, Direct Link 복사하여 사용
- **[imgur.com](https://imgur.com)** - 무료 이미지 업로드, Direct Link 복사하여 사용

## 데이터 저장 방식

- 모든 데이터는 브라우저의 **localStorage**에 저장됩니다.
- 처음 접속 시 `js/data.js`에 정의된 기본 데이터가 자동으로 로드됩니다.
- 관리자가 수정한 내용은 해당 브라우저에만 저장되므로, **다른 기기나 브라우저에서는 기본 데이터**가 표시됩니다.
- 브라우저 캐시/데이터를 삭제하면 수정 내용이 초기화됩니다.

### localStorage 키 목록

| 키 | 데이터 |
|----|--------|
| `academy_banners` | 배너 슬라이더 (배열) |
| `academy_instructors` | 강사진 (배열) |
| `academy_curriculum` | 교육과정 (배열) |
| `academy_notices` | 공지사항 (배열) |
| `academy_test_steps` | 테스트 절차 단계 (배열) |
| `academy_test_info` | 테스트 안내 정보 (객체) |
| `academy_contacts` | 연락처 (배열) |
| `academy_map_branches` | 캠퍼스 위치 정보 (배열) |
| `academy_map_info` | 대중교통/운영시간 (객체) |
| `academy_terms` | 이용약관 항목 (배열) |
| `academy_privacy` | 개인정보처리방침 항목 (배열) |

> 데이터를 초기화하려면 브라우저 개발자 도구(F12) → Application → Local Storage에서 해당 키를 삭제하면 됩니다.
